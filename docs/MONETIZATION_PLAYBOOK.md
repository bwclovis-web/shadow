# Monetization Playbook

Monetization should feel like acceleration for serious users, not friction for everyone else.

## Principles

- Preserve trust in ranking and recommendations
- Keep core value free and useful
- Make paid value obvious in under 10 seconds
- Label all sponsored/paid placements clearly

## Revenue Lanes

## 1) Shadows Plus (Subscription)

### Positioning

"For collectors and traders who want faster, smarter discovery."

### Candidate benefits

- Power filters and saved search bundles
- Priority alert delivery
- Advanced recommendation controls (season profile, note weighting)
- Monthly profile intelligence summary

### Pricing experiments

- [ ] **MZ-P-001** Monthly only: `$7.99`
- [ ] **MZ-P-002** Monthly + annual (2 months free)
- [ ] **MZ-P-003** Intro offer for first 30 days

## 2) Boosted Visibility (Trader side)

### Positioning

"Increase listing visibility for limited windows."

### Product guardrails

- Sponsored badge mandatory
- Capped boost density on each result page
- Relevance floor before boost eligibility

### Packages

- [ ] **MZ-B-001** 24h boost
- [ ] **MZ-B-002** 72h boost
- [ ] **MZ-B-003** Multi-listing bundle

## 3) Affiliate Commerce

### Positioning

"If a user cannot find a trade, show verified purchase alternatives."

### Placements

- In-stock alternatives on perfume detail
- "Best match available" module
- Seasonal editorial collections

### Trust requirements

- Explicit affiliate disclosure
- Prefer partners with return/refund clarity
- Avoid deceptive matching logic

## Funnel Design

### Entry points

- Profile page (upgrade panel)
- Saved search creation flow
- Alert settings
- Compare mode export/share

### Conversion steps

1. Trigger from high-intent action
2. Single-screen value explanation
3. Fast checkout
4. Immediate unlocked-state confirmation

## KPIs

- **Plus:** visitor-to-trial, trial-to-paid, churn, ARPU
- **Boosts:** trader attach rate, repeat purchase rate, message lift
- **Affiliate:** CTR, conversion proxy, revenue per 1k sessions

## Experiment Backlog

- [ ] **MZ-X-001** Paywall timing: immediate vs after repeated use
- [ ] **MZ-X-002** Package framing: "Pro tools" vs "Collector mode"
- [ ] **MZ-X-003** Boost CTA language and urgency
- [ ] **MZ-X-004** Affiliate module position on page

## Kill Criteria

- If paid placements lower trust signals (complaints, bounce, report rate) by defined threshold, pause and revise.
- If conversion is low and support burden rises, simplify offer before expanding scope.
